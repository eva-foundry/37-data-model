## Evidence Checklist

### Phase 1: Pre-Deployment

- [ ] **Local Tests Passing**: Screenshot of pytest output showing 100% pass
  - File: `pre-deployment/local-tests.png`
  - Expected: All tests green, exit code 0
  
- [ ] **Local Seed Successful**: JSON output of local seed operation
  - File: `pre-deployment/local-seed.json`
  - Expected: `{"total": N, "errors": []}`
  
- [ ] **Local Summary**: JSON output of local /model/agent-summary
  - File: `pre-deployment/local-agent-summary.json`
  - Expected: Layer counts populated

- [ ] **Docker Image Built**: Screenshot of docker images showing new tag
  - File: `pre-deployment/docker-image.png`
  - Expected: Tag matches deployment timestamp

### Phase 2: Azure Deployment

- [ ] **Container App Updated**: Screenshot of `az containerapp update` success
  - File: `deployment/aca-update.png`
  - Expected: Success message, revision name output

- [ ] **Revision Active**: Screenshot of `az containerapp revision list` showing new revision at 100%
  - File: `deployment/revision-active.png`
  - Expected: Latest revision with trafficWeight=100

- [ ] **Health Check Passing**: Screenshot of cloud /health endpoint response
  - File: `deployment/cloud-health.png`
  - Expected: `{"status": "ok", "store": "cosmos"}`
  - **CRITICAL**: Timestamp must be AFTER deployment (not from earlier session)

### Phase 3: Seed Operation (MOST CRITICAL)

- [ ] **Seed Request**: Curl/PowerShell command showing exact seed request
  - File: `seed/seed-request.txt`
  - Expected: `POST /model/admin/seed` with Bearer token
  
- [ ] **Seed Response JSON**: Complete JSON response from seed endpoint
  - File: `seed/seed-response.json`
  - Expected: `{"total": N, "errors": [], "duration": X}`
  - **CRITICAL**: total > 0, errors = []
  
- [ ] **Seed Response Screenshot**: Visual proof of seed completion
  - File: `seed/seed-screenshot.png`
  - Expected: Terminal/browser showing seed success
  - **CRITICAL**: Must show timestamp proving it's THIS deployment

- [ ] **Seed Operation Log**: PowerShell transcript or terminal log
  - File: `seed/seed-operation-log.txt`
  - Expected: Full command history from seed start to completion

### Phase 4: Cloud Verification

- [ ] **Cloud Summary JSON**: JSON output of cloud /model/agent-summary
  - File: `verification/cloud-agent-summary.json`
  - Expected: All layers showing data counts
  
- [ ] **Cloud Summary Screenshot**: Visual proof of cloud data
  - File: `verification/cloud-summary-screenshot.png`
  - Expected: Layer count and record count visible
  - **CRITICAL**: Numbers must match seed response

- [ ] **Sample Layer Query**: Query result from a major layer (e.g., wbs, endpoints)
  - File: `verification/sample-layer-query.json`
  - Expected: `GET /model/wbs/?limit=10` showing actual records

### Phase 5: Side-by-Side Comparison (MANDATORY)

- [ ] **Preflight Check**: JSON output from verification script phase 1
  - File: `comparison/01-preflight-check.json`
  - Generated by: `deployment-verification-sbs.ps1`
  - Expected: Both APIs reachable

- [ ] **Summary Comparison**: JSON output from verification script phase 2
  - File: `comparison/02-summary-comparison.json`
  - Generated by: `deployment-verification-sbs.ps1`
  - Expected: Layer counts match, record counts match

- [ ] **Layer-by-Layer**: JSON output from verification script phase 3
  - File: `comparison/03-layer-by-layer-comparison.json`
  - Generated by: `deployment-verification-sbs.ps1`
  - Expected: All layers show match=true

- [ ] **Evidence Pack**: Master evidence file from verification script
  - File: `comparison/EVIDENCE-PACK.json`
  - Generated by: `deployment-verification-sbs.ps1`
  - Expected: `verdict.status = "PASS"`

- [ ] **Verification Verdict Screenshot**: Screenshot of script final output
  - File: `comparison/verification-verdict.png`
  - Expected: "VERDICT: PASS" in green

### Phase 6: Documentation Update

- [ ] **Git Diff**: Screenshot of git diff showing documentation changes
  - File: `documentation/git-diff.png`
  - Expected: Only number changes, no aspirational claims

- [ ] **Git Commit**: Screenshot of git commit with verified numbers
  - File: `documentation/git-commit.png`
  - Expected: Commit message references verification timestamp

### Phase 7: Post-Deployment

- [ ] **Smoke Tests**: Screenshot of smoke test results
  - File: `post-deployment/smoke-tests.png`
  - Expected: All 5 critical endpoints responding

- [ ] **Deployment Record**: Screenshot showing deployment record created in API
  - File: `post-deployment/deployment-record.png`
  - Expected: Record with this timestamp exists in deployment_records layer
